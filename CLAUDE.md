# Дизайн-система портфолио

Живой документ. Все директивы по дизайну фиксируем здесь. Полигон — `src/pages/index.astro` (видимый скелет всех секций).

## Правила работы

- **Скорость > церемония.** Меньше TodoWrite, меньше пояснений, больше кода. Один Write на файл, файлы параллельно.
- **HMR работает сам.** Не пингую сервер, не перезапускаю без причины.
- **Фиксируем здесь** каждую утверждённую правку (палитра, типографика, токен, компонент, исключения).

## ЖЁСТКИЕ ПРАВИЛА (не нарушать)

1. **Текст:** на белом фоне — только чисто чёрный `#000000`; на синих секциях (Hero, Contact) — чисто белый `#ffffff`. Никакого серого, ни в каком виде.
2. **Никаких прозрачностей** (`rgba` с alpha, `opacity` на цветах, `color-mix`). Только сплошные тона. Исключение — тень пилюли-навбара.
3. **Заголовки — Inter 900 (black).** Всегда.
4. **Акцент — синий `#3B73B9`** (бывший оранжевый выкинут полностью). Используется как highlight-плашка и акцентные цифры/элементы.
5. **Бордеры — сплошные (solid).** Пунктир (dashed) запрещён — он читается как «черновик».
6. **Без нумерованных бирок** над секциями («02 / Работы» и т.п.) — убраны.
6a. **Без КАПСА.** `text-transform: uppercase` запрещён везде. Текст как написан.
7. Каждое новое дизайн-решение **сразу фиксируется здесь.**

## Шрифты

- **ТОЛЬКО Inter, два веса: Medium 500 и Black 900.** Больше ничего не грузим.
  - Body / метки / ссылки / **кнопки** → 500. Заголовки → 900. Токены весов — алиасы только на 500 или 900.
- **Моноширинный исключён** (`--font-mono` = алиас на `--font-sans`).
- **Разрядка (letter-spacing) = 0 везде.** Все tracking-токены `--tr-*` обнулены, literal-значения тоже.
- Display-шрифт не используем — Inter 900 тянет все заголовки.

### Works (галерея + фильтр) — ГЛАВНЫЙ БЛОК
Вдохновение — artlebedev.ru. **Сетка СНЯТА ЗАМЕРОМ с живого сайта** (chrome-devtools MCP, `getBoundingClientRect` на уже разложенном Vue-гриде `#als-portfolio-v3`). Числа ниже — реальные, не на глаз.

**Модель сетки artlebedev v5 (источник истины, замер на контейнере 1490px):**
- Ряд = `.p-v3-grid-line` (`display:block`); плитки = `inline-block; vertical-align:top` (по верху), ширина задана явно.
- **Юнитная сетка.** Два размера: `middle` = 2 юнита (740px ≈ 49.6%), `small` = 1 юнит (370px ≈ 24.8%). middle ровно ×2 от small.
- **Aspect-ratio (главное):** middle = 740/570 = **1.297** (ландшафт ~4:3); small = 370/570 = **0.649** (портрет ~2:3). База 1 юнита = `370/570 = 0.649`.
- В ряду плитки по верху; smalls добиты нижним отступом → внешние боксы выровнены ~570px → ряды стыкуются БЕЗ ДЫР.
- **Динамика по принципу контраста (наше решение):** сумма юнитов в ряду варьируется → разная высота рядов. Меньше сумма юнитов → шире юнит → ВЫШЕ ряд. У нас: ряд из 4 юнитов ≈532px, из **3** ≈713px (высокий, разреженный), из **5** ≈423px (низкий, плотный). Контраст задаём ОСОЗНАННО: размер (крупное × мелкое) + цвет (тёплый `#7C4A1E` × холодные синие). Текущие 4 ряда: `[m,s,s]`(4) · `[s,s,m]`(4) · `[m,s]`(3, высокий) · `[s,s,m,s]`(5, низкий).
- **Про «отступ первой карточки»:** реального отступа нет (всё на 15px). Это бенто-смещение швов между рядами (ряд `[m,s,s]` режет на 50/75%, ряд `[s,s,m]` на 25/50% — швы не совпадают). Это фича, а не баг; ритм держим осознанным через контраст.

**Наша реализация (`works.ts` + `Works.astro`):**
- У работы: `size: 'small'|'middle'` и `row`. Компонент группирует по рядам → `.row` (flex) → `.tile`.
- Плитка: `flex: var(--u) 1 0` (`--u` = юниты: small 1, middle 2) + `aspect-ratio: calc(var(--u) * 0.649)`. Т.к. AR ∝ юнитам, **все плитки ряда автоматически одной высоты** → дыр нет, fixed-высот рядов НЕ нужно. Проверено: рендер даёт middle 49.5%/AR 1.298, small 24.7%/AR 0.649 — копия оригинала.
- **Название ВНУТРИ плитки**, снизу (absolute), белым. Под ним **градиент** `linear-gradient(to top, surface 0% → color-mix 80% на 23% → transparent 83%)`. Снято с оригинала: у них `to bottom 0→0%(17%)→0.8(77%)→solid(100%)` в БРЕНД-ЦВЕТЕ плитки (инлайнится в каждую). Суть: картинка утопает в свой же цвет, плоская чистая зона сверху, solid (alpha 1) только у самого низа → белый текст читается при любой картинке. Исключение из «без прозрачностей» — градиент для читабельности разрешён.
- `surface` — насыщенный/тёмный цвет (бренд-цвет работы), он же красит градиент. Сейчас плейсхолдер вместо картинки/видео.
- **Посадка медиа — поле `fit` ('cover' | 'contain', по умолч. 'cover'):**
  - `cover` (дефолт) — full-bleed, обрезка под край. Под видео и скрины во всю плитку. Картинку готовить под AR плитки (middle 1.3 / small 0.65), экспорт 2× (middle 1480×1140, small 740×1140; под крупные мониторы — 2000×1540 / 1000×1540).
  - `contain` — **объект по центру, `surface`-цвет заливает свободные поля** (лого / мокап / прозрачный PNG). Реализация: `.tile[data-fit="contain"] .tile__fill { object-fit: contain; padding: 10% 10% 16% }` (низ — под зону градиента+подписи, поэтому padding-bottom больше). AR плитки для картинки тут не важен — объект вписывается как есть. Готовить: **прозрачный PNG/WebP**, длинная сторона 1200–1600px, с небольшим внутренним воздухом.
- **Плоско, без эффектов** (проверено: у artlebedev `transition: 0s`, нет hover-зума/тени/подъёма). У нас тоже убран `translateY` по ховеру — «живость» даёт сам медиа, не CSS.
- **База под видео:** `video` + `poster` → `<video muted loop playsinline preload="none">`, source `.mp4`, full-bleed cover. Автоплей **только в зоне видимости** (IntersectionObserver, threshold 0.25, preload включается лениво при входе) — как у Лебедева. `prefers-reduced-motion: reduce` → видео не запускается, остаётся `poster`. Гиф/статика — поле `image`. Приоритет медиа: video → image → только surface.
**Поля узкие, сайт полноширинный:** контейнер БЕЗ max-width (`--container-page: 100%`), боковые поля везде **15px**. Галерея — те же 15px, gap между плитками 8px (десктоп) / 6px (≤1079px).
Адаптив: AR тянет высоту от ширины сам (fixed-высот нет). **Мобилка ≤767px — модель artlebedev (проверено по их сайту):** `middle` во всю ширину (ландшафт, AR 1.5), `small` по **2 в ряд** (портрет, AR 0.66) — НЕ простая стопка. Реализация: `.works__grid` → `display:grid; grid-template-columns:1fr 1fr; grid-auto-flow:row dense`, `.row` → `display:contents` (плитки текут в общий грид сплошным потоком, small'ы парятся даже между исходными рядами; `dense` убирает дыры от одиночных), `middle` → `grid-column: span 2`. В гриде flex-свойства игнорируются, высоту даёт aspect-ratio (важно: раньше в flex-колонке `flex-basis:0` по вертикали схлопывал высоту в 0 → баг «белое место»).
Фильтр по категориям (`data-category`), активный чип — чёрный. **Заголовка «Работы» нет** — секция начинается сразу с чипов; чипы-плашки выровнены по центру (`justify-content: center`).
