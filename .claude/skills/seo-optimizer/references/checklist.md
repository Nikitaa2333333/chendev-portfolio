# Чек-лист SEO-аудита (Яндекс + Google)

Источник истины для аудита. Сгруппирован по темам и приоритету. Для каждого
пункта: **правило** → **как проверить** → **[АВТО]/[ВРУЧНУЮ]** → какому поисковику
важно. ~90% правил общие для обеих систем; яндекс- и google-специфика помечена.

Приоритеты: 🔴 высокий (блокеры индексации, включают сниппеты), 🟡 средний
(дубли, представление в выдаче), 🟢 низкий (полировка).

## Оглавление
1. Индексируемость и обход
2. Уникальность адреса (дубли)

## 1. Индексируемость и обход

- 🔴 **robots.txt существует и не блокирует лишнего.** Технические разделы
  (`/admin`, корзины, поиск по сайту) — `Disallow`. CSS/JS **не** закрывать:
  боты Яндекса и Google рендерят страницу и без стилей могут счесть её сломанной.
  → Проверить наличие файла и отсутствие `Disallow: /` или закрытых ассетов.
  → **[АВТО]**. Обе системы.
- 🔴 **sitemap.xml существует и указан в robots.txt** (`Sitemap: https://…`).
  Список всех индексируемых страниц в XML. → Проверить файл + строку в robots.
  → **[АВТО]**. Обе системы.
- 🔴 **Страницы отдают 200 OK; несуществующие — настоящий 404.** «Soft 404»
  (заглушка с кодом 200) заставляет поисковик держать мусор в индексе и медленнее
  обходить полезное. → Запросить заведомо несуществующий URL, проверить код.
  → **[ВРУЧНУЮ]** (нужен доступ к серверу/конфигу). Обе системы.
- 🟡 **Нет навигации только через JS/Flash.** Робот Яндекса не переходит по
  ссылкам, реализованным скриптами. Дублировать обычными `<a href>`. Для SPA —
  пре-рендер/SSR (см. stacks.md). → Проверить, что меню и ключевые ссылки — это
  `<a href>`. → **[ВРУЧНУЮ]** при SPA, иначе **[АВТО]**. Яндекс особенно.
- 🟡 **Без `<iframe>` для основного контента** — Яндекс не индексирует
  подгружаемое в iframe. → Греп по `<iframe`. → **[ВРУЧНУЮ]**.
- 🟢 **Документы < 10 МБ**, разумная длина URL без длинных списков cgi-параметров.

## 2. Уникальность адреса (дубли)

- 🔴 **Один протокол и один хост.** http→https и www↔без-www должны делать 301.
  Иначе поисковик видит до 4 копий сайта и размывает сигналы. → Открыть все
  варианты **на проде** и проверить редирект (`curl -sI`, см. deploy-and-prod.md).
  → **[ВРУЧНУЮ]** (`.htaccess`/панель хостинга; нюанс прокси-SSL — там же). Обе.
  Для Яндекса 301 мало — дополнительно подать заявку «Переезд сайта» в Вебмастере
  для склейки зеркал (см. manual-steps.md). Яндекс.
- 🔴 **У каждой страницы один путь доступа** — без дублей через альтернативные
  URL. Частые дубли: `/index.html` ↔ `/`, со слешем ↔ без, регистр. → Свести 301-м
  к одному адресу + внутренние ссылки вести на канонический. Реализация зависит
  от стека (статика/Apache — `.htaccess`; Nginx — конфиг; Next/CMS — роутер/
  настройки; см. stacks.md). Проверить `curl -sI` по вариантам. → **[ВРУЧНУЮ/АВТО]**.
- 🔴 **`<link rel="canonical">` на каждой странице** с абсолютным каноническим
  URL. Защищает от дублей по параметрам/слешам. → Проверить наличие и абсолютность.
  → **[АВТО]**. Обе системы.
- 🟡 **Без идентификаторов сессий в URL**, один контент — один адрес.
  → **[ВРУЧНУЮ]**.
- 🟡 **Избегать лишних редиректов** (цепочек). → **[ВРУЧНУЮ]**.

## 3. Метаданные `<head>`

- 🔴 **`<title>` — уникальный, осмысленный, с ключевым запросом и гео.** Это
  ссылка на сайт в выдаче, «визитка». Ориентир 50–60 знаков. Пример:
  «Типография в Москве — печать полного цикла | СпектрГрупп». → Есть ли тег, не
  дублируется ли между страницами, содержит ли суть+регион. → **[АВТО]** (текст
  согласовать). Обе системы.
- 🔴 **`<meta name="description">` — уникальное, 120–160 знаков, с выгодой и
  призывом.** Часто становится сниппетом. → Наличие + длина + уникальность.
  → **[АВТО]**. Обе системы.
- 🔴 **`<meta name="viewport" content="width=device-width, initial-scale=1">`** —
  сигнал мобилопригодности. → Наличие. → **[АВТО]**. Обе системы.
- 🟡 **`<html lang="ru">`** — язык страницы. → Атрибут есть. → **[АВТО]**.
- 🟡 **Open Graph** (`og:title`, `og:description`, `og:image`, `og:url`,
  `og:type`) — Яндекс их поддерживает для сниппета, плюс красивые ссылки в
  мессенджерах/соцсетях. `og:image` ≥ 1200×630. → Наличие набора. → **[АВТО]**.
  Обе системы.
- 🟡 **Twitter Card** (`twitter:card=summary_large_image` + title/description/
  image) — превью в X и части агрегаторов. → **[АВТО]**.
- 🟢 **Фавикон** в формате, который Яндекс поддерживает для выдачи (по их доке):
  **SVG (рекоменд.), ICO, PNG, GIF, JPEG, BMP — `webp` НЕ поддерживается**, потому
  webp-фавикон даёт в выдаче пустой глобус. Размер 120×120 (или SVG), файл
  `favicon` в корне + `<link rel="icon">` (можно и `apple-touch-icon`). Одинаковый
  по всем адресам (www/https). В выдаче обновляется ~2 недели после обхода.
  → Проверить формат и `<link rel="icon">`. → **[АВТО]** (разметка) + **[ВРУЧНУЮ]**
  (сгенерировать .ico/.png/.svg, если исходник в неподдерживаемом формате).

## 4. Микроразметка (структурированные данные) — JSON-LD

Самый высокоценный невидимый слой. Шаблоны — в `schema-templates.md`. Формат —
JSON-LD в `<script type="application/ld+json">` (поддерживают и Яндекс, и Google).

- 🔴 **`Organization` или `LocalBusiness`** — название, лого, контакты, адрес,
  часы работы, гео. Включает **адресный сниппет** в Яндексе (нужен Топ-30 + та же
  инфа в Яндекс.Бизнесе) и панель знаний у Google. → Есть ли JSON-LD с типом.
  → **[АВТО]**. Обе системы.
- 🔴 **`FAQPage`** — если на странице есть блок вопрос-ответ. Даёт раскрытый
  FAQ-сниппет прямо в выдаче. → Есть FAQ в DOM, но нет разметки. → **[АВТО]**.
  Обе системы.
- 🟡 **`BreadcrumbList`** — навигационные цепочки в сниппете; для многостраничных.
  → **[АВТО]** на многостраничнике.
- 🟡 **`WebSite`** (+ `SearchAction` при наличии поиска по сайту). → **[АВТО]**.
- 🟢 **`Product` / `Article` / `Service`** — по тематике. → **[АВТО]** если
  подходит.
- Проверять разметку валидаторами (см. manual-steps.md). Ошибочную Яндекс просто
  пропускает — но лучше чисто.

## 5. Контент и семантика

- 🔴 **Одна `<h1>` на страницу + иерархия `<h2>…<h6>`.** Помогает и пользователю,
  и роботу понять структуру. → Ровно одна h1, заголовки по смыслу. → **[АВТО]**
  (если ломано) / **[ВРУЧНУЮ]** (переписать). Обе системы.
- 🔴 **Семантические лендмарки**: `<header> <nav> <main> <section> <footer>`.
  → Наличие `<main>` особенно. → **[АВТО]**.
- 🔴 **Достаточно индексируемого текста, отвечающего на запросы.** Страница из
  одних картинок/тонкий текст рискует статусом «малоценная/маловостребованная» у
  Яндекса. Под целевые запросы (Wordstat) — упомянуть термины в тексте. →
  Оценить объём и релевантность. → **[ВРУЧНУЮ]** (контент-решение). Обе системы.
- 🟡 **Контент доступен без действий** (без обязательной регистрации/SMS, чтобы
  увидеть текст) — иначе «глубинный веб», робот не видит. → **[ВРУЧНУЮ]**.
- 🟡 **Грамотность и аккуратное оформление**, без переоптимизации (спама
  ключами) и агрессивной рекламы — Яндекс штрафует за низкое юзабилити. →
  **[ВРУЧНУЮ]**.
- 🟢 **Осмысленные тексты ссылок** (не «тут»/«рис.1»), внутренняя перелинковка.

## 6. Изображения

- 🔴 **`alt` у всех `<img>`** — для доступности и Яндекс/Google Картинок.
  → Доля img без alt. → **[АВТО]**. Обе системы.
- 🟡 **Не подменять текст картинкой.** Текст на изображении не индексируется —
  дублировать его обычным текстом рядом. → **[ВРУЧНУЮ]**.
- 🟡 **Осмысленные имена файлов** (`vizitki-premium.webp`, не `img_1234.jpg`).
  → **[ВРУЧНУЮ]** (требует переименования + правок ссылок).
- 🟢 **Современный формат** (webp/avif), сжатие, корректные размеры; декоративные/
  служебные картинки (счётчики, CAPTCHA) — закрыть в robots.

## 7. Мобильность

- 🔴 **Адаптивный дизайн, один URL для всех устройств** — предпочтительная схема
  для обеих систем. Контент не вылезает за экран по горизонтали при ширине
  320px+. → `viewport` + ресайз-проверка. → **[АВТО]** (viewport) / **[ВРУЧНУЮ]**
  (вёрстка). Обе системы.
- 🟡 **Если отдельная мобильная версия (m.example.com)** — связать редиректом
  (для мобильного робота) + `rel="alternate"` (для основного робота) на десктопе.
  → **[ВРУЧНУЮ]**. Яндекс.
- 🟡 **Без Flash/Silverlight/Applet** на мобильных страницах. → **[ВРУЧНУЮ]**.

## 8. Скорость / Core Web Vitals

- 🔴 **LCP-картинка предзагружена** (`<link rel="preload" as="image"
  fetchpriority="high">`) и в современном формате. → Наличие. → **[АВТО]**.
  Google (CWV — фактор), Яндек (юзабилити).
- 🟡 **Картинки ниже первого экрана — `loading="lazy"`**; у `<img>` заданы
  `width`/`height` (защита от CLS). → **[АВТО]**.
- 🟡 **Кэширование статики** (`Cache-Control: immutable` для хэшированных
  ассетов), сжатие (gzip/brotli). → **[ВРУЧНУЮ]** (`.htaccess`/хостинг).
- 🟢 Минимизация рендер-блокирующих ресурсов; не грузить 4K в маленький блок.
  Замерять реальным инструментом (PageSpeed Insights / Lighthouse).

## 9. Аналитика и панели вебмастера

- 🔴 **Счётчик аналитики на ВСЕХ страницах.** Яндекс.Метрика и/или Google
  Analytics в `<head>` каждой страницы (частая ошибка — счётчик только на
  главной). Метрика — ещё и поведенческий сигнал для Яндекса. → Грепнуть тег по
  всем страницам. → **[АВТО]** (вставить недостающие). Обе системы.
- 🔴 **Подтверждение прав в Вебмастере + Search Console.** → Проверить наличие
  verification-файла/метатега. → **[ВРУЧНУЮ]** (нужен аккаунт; см.
  manual-steps.md). Соответственно Яндекс / Google.
- 🟡 **Sitemap отправлен в обеих панелях**, настроен регион (Яндекс) и основное
  зеркало/хост. → **[ВРУЧНУЮ]**.
- 🟡 **Обход по счётчику аналитики** (ускоритель индексации). Яндекс умеет
  обходить страницы по данным Метрики: привязать счётчик к Вебмастеру и включить
  «Обход по счётчикам». Особенно помогает молодым сайтам. → **[ВРУЧНУЮ]**. Яндекс.
- 🟢 **IndexNow** — мгновенный пинг «обойди URL» (Яндекс + Bing; Google не
  поддерживает). Ключ-файл в корне + запрос на их API. → **[ВРУЧНУЮ/АВТО]**.
- 🟢 **Подключить Яндекс.Бизнес** для адресного сниппета и карточки на Картах.
  → **[ВРУЧНУЮ]**. Яндекс.

> **Индексация ≠ мгновенно.** После чистой технички попадание в индекс — на
> цикле обновления поисковика. «Обойдено 200, но нет в базе» — нормальный статус
> очереди, не баг. Внутренние страницы часто индексируются раньше главной.
> Подробнее (ускорители, сроки, случай «индексируется не тот URL») —
> deploy-and-prod.md.
