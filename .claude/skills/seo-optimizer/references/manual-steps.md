# Задачи только для человека — точные инструкции

Это пункты [ВРУЧНУЮ]: нужны аккаунты, доступ к серверу или решения по контенту.
Скилл их не выполняет, но обязан дать пользователю пошаговую инструкцию и точные
ссылки. Ниже — готовый материал; адаптируй под проект.

## 1. Подтверждение прав в Яндекс.Вебмастере

1. Зайти на https://webmaster.yandex.ru/ → «Добавить сайт» → ввести
   `https://<домен>`.
2. Выбрать способ подтверждения. Для статики проще **HTML-файл**: Яндекс даёт
   файл вида `yandex_xxxxx.html` — положить в корень сайта, задеплоить, нажать
   «Проверить». (Альтернатива — метатег в `<head>` главной.)
3. После подтверждения: указать **регион** сайта, проверить главное зеркало.

## 2. Подтверждение прав в Google Search Console

1. https://search.google.com/search-console → «Добавить ресурс».
2. Тип **«Ресурс с префиксом в URL»** → `https://<домен>/`.
3. Способ — HTML-файл (`google*.html` в корень) или метатег. Задеплоить →
   «Подтвердить».
4. Можно подтвердить **домен целиком** через DNS TXT-запись (покрывает все
   протоколы/поддомены) — если есть доступ к DNS.

## 3. Отправить sitemap в обе панели

- Яндекс.Вебмастер → «Индексирование» → «Файлы Sitemap» → добавить
  `https://<домен>/sitemap.xml`.
- Search Console → «Файлы Sitemap» → добавить `sitemap.xml`.
- Можно вручную отправить ключевые страницы на переобход (Вебмастер →
  «Переобход страниц»; GSC → «Проверка URL» → «Запросить индексирование»).

## 4. Аналитика: Яндекс.Метрика и Google Analytics

Сам код счётчика [АВТО] вставляется в `<head>` **всех** страниц, но **номер
счётчика и аккаунт создаёт человек**:
- Метрика: https://metrika.yandex.ru/ → создать счётчик → взять `id` → вставить
  в код (включить Вебвизор/карту кликов по желанию). Ставить как можно ближе к
  началу `<head>`.
- GA4: https://analytics.google.com/ → создать ресурс → взять `G-XXXX` → вставить
  тег `gtag.js`.
- Проверить, что счётчик есть на **каждой** странице (частая ошибка — только
  главная).

## 5. Подбор целевых запросов (Wordstat)

Чтобы title/description/тексты отвечали реальным запросам:
1. https://wordstat.yandex.ru/ → ввести услугу/товар (напр. «печать визиток
   Москва») → «Подобрать».
2. Слева — частотности с этим словом, справа — связанные запросы (синонимы,
   уточнения). Числа = показов/мес.
3. Выбрать 3–5 частотных релевантных фраз, выставить регион. Вплести их
   естественно в title, h1, description и основной текст — **без переспама**.
4. Отслеживать: Вебмастер → «Поисковые запросы»; Метрика → отчёт «Поисковые
   запросы».

## 6. HTTPS и редиректы (один адрес сайта)

Нужен доступ к серверу/хостингу. Цель — чтобы http→https и www↔без-www делали
**301**, иначе поисковик видит дубли сайта.

Apache (`.htaccess`), вариант «https + без www»:
```apache
RewriteEngine On
# http -> https
RewriteCond %{HTTPS} off
RewriteRule ^ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
# www -> без www
RewriteCond %{HTTP_HOST} ^www\.(.+)$ [NC]
RewriteRule ^ https://%1%{REQUEST_URI} [L,R=301]
```
Nginx — аналогично через `return 301`. На панельных хостингах часто есть
переключатель «Принудительный HTTPS» и «Главный домен». SSL-сертификат — обычно
бесплатный Let's Encrypt в панели. За прокси (SSL терминируется не на Apache)
вместо `%{HTTPS} off` используй `%{HTTP:X-Forwarded-Proto} !https` —
см. deploy-and-prod.md.

**После настройки редиректов — заявка на «Переезд сайта» (важно для Яндекса).**
Яндекс считает `http`/`https`/`www`/без-www **разными сайтами**, которые
конкурируют в выдаче; одних 301 мало, чтобы он быстро понял главное зеркало.
1. Добавь в Вебмастер **оба** адреса (напр. `http://` и `https://`, или с www и
   без) и подтверди права на каждом.
2. Убедись, что 301-редирект со старого на главный работает (проверь `curl -sI`).
3. Вебмастер → **Индексирование → Переезд сайта** → отправь заявку (старый →
   новый/главный адрес). Это ускоряет склейку зеркал и показывает ошибки.
4. Прогресс смотри в **Индексирование → Страницы в поиске** (старый адрес уходит,
   главный появляется). Обратная операция — «Разгруппировка сайтов».
Симптом, что склейка не завершена: в Вебмастере у страниц статус «данные о
каноническом адресе обрабатываются».

## 7. Кастомная 404 и борьба с soft 404

- Несуществующие URL должны отдавать **код 404** (а не 200 с заглушкой).
- Сделать понятную страницу 404 со ссылкой на главную/каталог.
- Apache: `ErrorDocument 404 /404.html`. Проверить кодом ответа (Вебмастер →
  «Проверка ответа сервера», или `curl -I`).

## 8. Контент: «тонкие» и важные правки

- Если страница почти без текста (только картинки) — добавить осмысленный текст
  под целевые запросы, иначе риск «малоценной» у Яндекса.
- Текст с картинок продублировать обычным текстом.
- Убрать агрессивную рекламу/попапы, перекрывающие контент (минус к юзабилити).
Это редакторские решения — согласовать с владельцем сайта.

## 9. Проверка результата

- Микроразметка: Яндекс — «Валидатор микроразметки» (в Вебмастере); Google —
  Rich Results Test (https://search.google.com/test/rich-results).
- Мобилопригодность: Вебмастер → «Проверка мобильных страниц».
- Скорость: PageSpeed Insights (https://pagespeed.web.dev/) — даёт и
  лабораторные, и полевые Core Web Vitals.
- Появление в выдаче/сниппеты — недели, не дни. Фавикон и разметка у Яндекса —
  обычно до ~2 недель после обхода.
